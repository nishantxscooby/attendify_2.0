# Contributing to Attendify 2.0


Thanks for your interest in contributing! We welcome improvements, bug fixes, translations, and documentation.


## How to contribute


1. Fork the repository.
2. Create a feature branch: `git checkout -b feat/your-feature`.
3. Commit changes with descriptive messages.
4. Push to your fork and open a Pull Request describing the change.


## Coding standards
- Follow existing code style (PEP8 for Python, Prettier/ESLint for JS/TS).
- Add tests for new features where possible.


## Pull request process
- PRs should target `main` (or the current stable branch).
- Include a clear description, motivation, and screenshots if UI-related.
- Link relevant issues with `Fixes #<issue-number>` where appropriate.


## Local setup
Include any local dev setup here (see project README for per-component steps).


## Security
If you discover a security vulnerability, please **do not open a public issue**. Instead, email the maintainer at `officialnishant21@gmail.com` with details.
